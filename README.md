\# AI基础对话机器人AI 基础对话机器人（通义千问 API 无依赖版）
项目简介
本项目是一款基于阿里云通义千问 qwen-turbo 大模型开发的本地 AI 对话机器人，采用 Python 内置标准库实现网络请求，无需额外安装第三方依赖库，降低运行门槛。
项目集成三大核心功能：多轮上下文对话、AI 角色切换、对话记录查看与清空，同时支持对话内容本地持久化保存，适合 Python 入门与大模型 API 调用学习，可作为课程实践作业使用。
一、运行环境要求
操作系统：Windows /macOS/ Linux 全平台兼容
Python 版本：Python 3.7 及以上（系统自带标准库，无需额外安装包）
网络要求：设备可正常访问公网（用于调用阿里云通义千问 API 接口）
必备凭证：阿里云通义千问 API Key（sk- 开头密钥）
二、前置准备：获取 API Key
登录阿里云百炼 / 灵积控制台，完成账号实名认证（个人免费使用必备）。
左侧导航栏进入 API-KEY 管理，点击创建 API-KEY，权限选择「全部」。
复制生成的 sk- 开头密钥（密钥仅创建时完整展示，务必提前保存）。
三、项目配置步骤
克隆 / 下载本项目到本地，进入项目根目录，找到 main.py 文件。
打开 main.py，定位到配置行，将代码中默认的 API_KEY 替换为你自己的密钥：
python
保存文件（Ctrl + S），配置完成。
四、项目启动与运行
方式 1：终端运行（推荐）
打开 CMD / PowerShell / Git Bash，切换到项目根目录（路径示例）：
powershell
cd D:\ai-chat-bot
执行运行命令：
powershell
python main.py
程序启动成功后，终端会展示功能说明，出现 你： 提示符即可开始交互。
方式 2：编辑器直接运行
使用 VS Code、IDLE 等编辑器打开 main.py，点击「运行」按钮直接启动程序。

八、常见问题与故障排查
请求失败：Invalid API Key
问题原因：API 密钥填写错误、存在空格 / 换行，或密钥权限异常。
解决方案：重新核对 main.py 中的密钥，确保为完整 sk- 字符串，无多余符号。
请求超时 / 无响应
问题原因：网络不通、阿里云控制台地域错误（需设置为华北 2（北京））。
解决方案：检查本地网络，登录阿里云控制台确认地域配置。
Python 不是内部或外部命令
问题原因：Python 未添加到系统环境变量。
解决方案：重新安装 Python，勾选 Add Python to PATH，或使用 py main.py 运行。
无法生成 chat_log.txt 文件
问题原因：项目文件夹权限不足。
解决方案：将项目移动到桌面 / 非系统盘目录，重新运行程序。
九、安全规范与注意事项
密钥安全（重点）：API_KEY 属于私密凭证，禁止直接上传明文代码到 GitHub、码云等公开仓库，避免密钥被盗用产生额外费用。
免费额度：阿里云新用户提供免费 Token 额度，满足日常学习与作业使用，额度耗尽后需自行充值。
接口限制：请勿高频、恶意调用 API 接口，遵守阿里云大模型接口使用规范。
十、技术说明
本项目未使用 dashscope 等第三方 SDK，基于 Python 内置标准库实现：
http.client：原生 HTTPS 请求，调用通义千问官方 API 接口
json：实现请求体与响应数据的序列化、反序列化
纯标准库方案彻底解决虚拟环境、依赖安装失败等问题，适配各类 Python 运行环境。
